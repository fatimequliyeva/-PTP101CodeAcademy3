# 🚀 Product & Category API (Node.js + MongoDB)

Bu layihə Node.js, Express və MongoDB (Mongoose) istifadə edilərək hazırlanmış RESTful API-dir və məqsədi MVC arxitekturasının tətbiqi, MongoDB modellərinin qurulması, middleware istifadəsi və modellər arasında One-to-Many münasibətinin praktiki şəkildə göstərilməsidir. Layihədə iki əsas model mövcuddur: Category və Product. Category modeli `name` (required) və `description` sahələrindən ibarətdir, Product modeli isə `title` (required), `description`, `price` (required), `stock` (default: 0) və `category` (ObjectId, ref: Category) sahələrini ehtiva edir və bu sahə vasitəsilə hər məhsul müəyyən bir kateqoriyaya bağlanır. API vasitəsilə kateqoriyalar və məhsullar üçün tam CRUD əməliyyatları həyata keçirilir; `/api/categories` endpointi bütün kateqoriyaları gətirmək, tək kateqoriyanı əldə etmək, yeni kateqoriya yaratmaq, mövcud kateqoriyanı yeniləmək və silmək funksiyalarını təmin edir, `/api/products` endpointi isə məhsullar üçün eyni CRUD əməliyyatlarını yerinə yetirir və əlavə olaraq query parametrləri vasitəsilə axtarış və sıralama imkanları təqdim edir (məsələn `?search=iphone` və `?sort=asc/desc`). Məhsulun tək gətirilməsi zamanı `populate` istifadə edilərək məhsula aid olan kateqoriya məlumatları da birlikdə qaytarılır. Layihədə bir neçə middleware istifadə edilmişdir: `loggerMiddleware` bütün gələn sorğuların metodunu və endpointini console-da göstərərək request izlənməsini təmin edir, `categoryValidator` və `productValidator` isə göndərilən məlumatların doğruluğunu yoxlayır (məsələn `title` boş ola bilməz və `price` mənfi ola bilməz) və səhv olduqda `400 Bad Request` statusu qaytarır. Bundan əlavə API-nin təhlükəsizliyini və stabilliyini artırmaq üçün `express-rate-limit` istifadə edilərək müəyyən zaman intervalında göndərilə bilən request sayı məhdudlaşdırılmışdır və bu yanaşma spam requestlərin və mümkün DDoS tipli yüklənmələrin qarşısını almağa kömək edir. Layihədə express, mongoose, dotenv, cors, nodemon və express-rate-limit paketlərindən istifadə edilmişdir və server `.env` faylında saxlanılan `PORT` və `MONGO_URI` dəyişənləri vasitəsilə konfiqurasiya olunur; layihəni işə salmaq üçün əvvəlcə `npm install` ilə asılılıqlar yüklənir, sonra `.env` faylı yaradılır və sonda `npm run dev` və ya `nodemon server.js` komandası ilə server başladılır.
